# 安全政策

追番中心会访问本地媒体文件、SQLite 数据库、RSS、外部元数据服务和受管 qBittorrent。安全问题可能影响用户的本地文件、下载任务或桌面会话，请通过私密渠道提交可利用细节。

## 支持范围

安全修复面向 `main` 分支和最新发布版本。旧版本不再接收安全更新；报告问题时请附上版本号或 commit。

以下问题适合提交安全报告：

- 桌面一次性会话令牌、鉴权绕过或跨来源泄露。
- 任意文件读取、写入、路径穿越或目录选择边界失效。
- qBittorrent 凭据、端口、任务控制或本地 API 暴露。
- 图片代理、RSS、元数据抓取产生的 SSRF、重定向绕过或载荷校验缺陷。
- 安装包、自动启动链路、依赖或随包二进制的供应链问题。
- 可能泄露数据库、日志、播放记录、下载记录或其他个人数据的问题。

## 如何报告

优先使用 GitHub 的 Private vulnerability reporting：

1. 打开仓库的 **Security** 页面。
2. 进入 **Advisories**。
3. 选择 **Report a vulnerability**，提交受影响版本、影响、复现条件和建议修复。

如果该入口暂时不可用，可以创建标题以 `[Security]` 开头的普通 Issue。公开 Issue 只写非敏感摘要、受影响版本和可联系状态，等待维护者建立私密沟通渠道。

请勿在公开 Issue、讨论、日志或截图中提交：

- 密码、令牌、Cookie、API Key 或私钥。
- 可直接执行的完整利用代码。
- 真实 magnet、tracker、下载任务、文件路径或个人媒体库内容。
- 未经遮盖的数据库、配置文件和 `%APPDATA%\anime-tracker` 日志。

## 安全研究边界

- 仅在你拥有或明确获准使用的设备、账号和数据上验证。
- 使用合成数据、假磁力链接和隔离目录构造复现。
- 避免访问、修改或删除他人的数据。
- 避免让测试继续上传、做种或产生无关网络流量。
- 上游组件的问题也应报告给对应项目；若追番中心的集成方式放大了影响，请同时在这里提交集成层报告。

维护者确认问题后会评估影响版本、准备修复并协调披露时间。在修复公开前，请给用户保留合理的更新窗口。
