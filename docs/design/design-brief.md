# 追番中心 — Design Brief

> 版本：v1.0 | 日期：2026-05-24
> 基于 /shape Discovery Interview 产出

## 1. Feature Summary

个人追番管理网站，核心解决"今天看什么、看到第几集、下集什么时候更新"。兼顾每季选番、观看进度管理、RSS 自动下载管理。视觉定位是带有二游官网 / 番剧官网气质的个人媒体中心。

## 2. Primary User Action

打开首页 → 3 秒内知道今天该看什么、哪些更新了、上次看到哪集。

## 3. Design Direction

### 视觉气质
"深色影厅 + 玻璃悬浮" — 参照 Obsidian OS 的岩石质感深底 + Anime.to 的全出血主视觉 + Pitch 的 3D 空间感。界面本身极度克制、几乎无色，让番剧封面和海报成为唯一的色彩来源。

### 视觉参考

| 参考 | 取什么 |
|------|--------|
| Anime.to | 全出血主视觉 Hero，文字叠在画面上，深色调底 |
| Pitch | 3D 透视卡片排列，极简排版，大留白，空间感 |
| Obsidian OS | 深色岩石质感背景，玻璃拟态悬浮卡片，暖灰色调 |

参考图位置：`docs/design/references/`

### 配色系统

| 层级 | 色值方向 | 说明 |
|------|----------|------|
| 背景 | `#0a0a0b` → `#141416` | 近黑，微暖灰底 |
| 表面 | `rgba(255,255,255,0.04~0.08)` | 玻璃卡片层，backdrop-blur |
| 边框 | `rgba(255,255,255,0.06~0.10)` | subtle border |
| 文字 | `#e8e8e8` / `#888888` | 主文字 / 次要文字 |
| 强调色 | 主题级 token + 详情页封面局部覆盖 | 全站主行动、选中态、进度条、orbit ring、hover halo 必须引用同一组 accent token |

### 全局氛围模式

| 模式 | 色温 | 参考气质 |
|------|------|----------|
| 潮流 | 高对比 + 赤红暖调 | ZZZ 的街头感 |
| 治愈 | 低对比 + 暖米色调 | Frieren 的柔光 |
| 复古 | 中对比 + 偏红暖紫 | Reverse:1999 的质感 |
| 蜜桃 | 柔和 + 蜜桃粉 | 轻甜但保持暗色克制 |
| 科幻 | 高对比 + 冷青调 | 冷调但不是蓝紫粉 |

进入番剧详情页时，叠加该番封面的提取色，覆盖全局强调色。

外观切换不能只换浅层背景色。每个外观模式都必须有自己的 `--accent`、`--accent-rgb`、`--accent-muted`、`--accent-subtle`、`--accent-contrast`，并让卡片动效、导航选中态、按钮、进度条、微光统一变化。蓝色可以作为主题色；紫色主题只走偏红暖紫，避开蓝紫倾向、蓝紫粉组合/渐变和廉价渐变。主题 accent 对深色背景、accent contrast 对实心按钮均需满足 WCAG AA 4.5:1。

### 排版

- 标题：大字重 700-800，紧字距 `letter-spacing: -0.02em`
- 正文：14-16px，`line-height: 1.6`
- 数据数字：`font-variant-numeric: tabular-nums`
- 字体：英文 Inter (variable) / 中文 Noto Sans SC

### 质感

- Noise texture overlay（opacity 0.02-0.03）
- Backdrop-blur 10-20px
- 阴影：`0 8px 32px rgba(0,0,0,0.3)`
- Border：`1px solid rgba(255,255,255,0.06)`
- Hero 区域 gradient mesh 光晕
- 精制微光可以用（低透明度 halo、ambient glow、柔光晕染）
- 禁止廉价发光（高饱和度大范围 glow、霓虹描边、彩色 box-shadow spread）

## 4. Layout Strategy

### 首页信息流

```
┌─────────────────────────────────────────────────┐
│  [Nav] Logo    搜索    通知    头像               │
├─────────────────────────────────────────────────┤
│  ┌─ Hero Spotlight ──────────────────────────┐  │
│  │  全出血番剧海报，渐变遮罩                      │  │
│  │  番名 / 简介 / 评分 / 最新集                  │  │
│  │  [继续观看] [详情]    ← → 轮播控制            │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  ── 今日更新 ──  横向滚动卡片行                    │
│  ── 继续观看 ──  横版缩略 + 进度条                 │
│  ── 本季新番 ──  按星期分组，超出首屏可横滑          │
│  ── 漏看提醒 ──  有更新但未看的番                  │
└─────────────────────────────────────────────────┘
```

视觉权重：Hero 60% / 今日更新 20% / 其余 20%

### 番剧详情页

```
┌─────────────────────────────────────────────────┐
│  全屏 Key Visual Hero + 渐变遮罩底部信息           │
│  番名 / 评分 / 类型标签 / [追番] [下载]            │
├─────────────────────────────────────────────────┤
│  Tab: 简介 | 制作组 | 声优                       │
│  集数列表: ✓已看 ●当前 ○未看 ○未播出               │
│  我的评分 + 笔记                                  │
└─────────────────────────────────────────────────┘
```

## 5. Key States

| 状态 | 用户看到什么 | 用户感受 |
|------|------------|---------|
| 首次使用（空库） | 引导："添加你的第一部番剧" + 搜索框 | 简单、不迷茫 |
| 日常打开（有数据） | Hero + 今日更新 + 继续观看 | 3 秒找到想看的 |
| 今日无更新 | "今天休息日。明天有 X 部更新。" | 不失落，有期待 |
| 搜索中 | 骨架屏 + 微妙呼吸动效 | 不焦虑 |
| API 不可用 | 缓存数据 + 顶部提示条 | 不阻断使用 |
| 下载中 | 卡片角标进度 | 一眼知道状态 |
| 下载失败 | 角标变红 + 失败原因 + 重试 | 知道怎么处理 |
| 追番满载 50+ | 列表/网格切换 + 筛选排序 | 不混乱 |

## 6. Interaction Model

### 微交互（Motion / Framer Motion）

| 触发 | 效果 |
|------|------|
| 卡片 hover | translateY(-4px) + scale(1.02) + 阴影加深, 0.2s ease-out |
| 页面进入 | stagger fade-in, 间隔 50ms, 0.4s |
| 滚动 | scroll reveal, y:20px opacity:0 → visible |
| 本季新番横向轨道 | 每个星期首屏 6 张，超出后左右箭头 / 触控板横滑，卡片 snap 对齐 |
| 追番状态切换 | 按钮形变 + 颜色过渡, 0.3s |
| 集数标记 | checkmark draw animation, 0.3s |
| Hero 轮播 | crossfade, 0.6s |
| 页面切换 | layout animation, 共享封面过渡 |

### 3D 背景（首页 + 登录页）
- React Three Fiber particle field
- 粒子 < 200, opacity 0.3-0.5
- 颜色从 Hero 封面取
- 缓慢漂浮 + 鼠标视差
- 降级：prefers-reduced-motion → CSS radial-gradient

### 关键流程

**标记已看**：详情页 → 点集数 → 标记 → 进度更新 → 看完本季 → 提示"移到已看完？"

**追新番**：新番页 → 网格浏览 → 半屏预览 → "追番" → 选状态 → 回列表，卡片显示已追

## 7. Content Requirements

| 位置 | 内容 |
|------|------|
| 空库引导 | "添加你的第一部番剧" + 搜索框 |
| 今日无更新 | "今天休息日。明天有 X 部更新。" |
| 追番状态 | 在看 / 想看 / 看完 / 搁置 / 弃番 |
| 下载状态 | 等待 / 下载中 XX% / 已完成 / 失败 |
| API 离线 | "数据源暂时不可用，显示的是缓存数据" |
| 集数未播 | "EP.13 预计 6月7日 播出" |

### 数据范围

| 数据 | 最少 | 典型 | 最多 |
|------|------|------|------|
| 追番数量 | 1 | 15-30 | 100+ |
| 单番集数 | 1 | 12-24 | 500+ |
| 今日更新 | 0 | 3-5 | 10+ |
| RSS 源 | 1 | 3-5 | 10 |

## 8. 核心组件

1. **AnimeCard** — 封面+标题+进度+状态（竖版/横版）
2. **GlassPanel** — 通用玻璃拟态容器
3. **HeroSpotlight** — 全出血轮播
4. **EpisodeGrid** — 集数网格（✓●○🔒）
5. **StatusBadge** — 追番状态切换器
6. **ProgressBar** — 观看进度条（颜色从番剧强调色取）
7. **SearchCommand** — Cmd+K 全局搜索

## 9. Open Questions

| 问题 | 建议 |
|------|------|
| 封面色提取算法 | colorthief 或 vibrant.js，取 dominant + muted |
| 搜索策略 | 本地 SQLite 优先，搜不到 fallback API |
| Bangumi 交叉匹配失败 | 保留日文名 + 手动编辑入口 |
| qBit 认证 | 自带 cookie 认证，走 localhost |
| 低配降级 | prefers-reduced-motion + GPU 检测 → CSS fallback |

## 10. 设计禁止项

- 蓝紫色、蓝紫粉三色组合、蓝紫渐变和廉价渐变
- 只改变背景 / surface / border，却不替换 accent token 的“半套主题切换”
- 廉价发光（高饱和 glow、霓虹、彩色 shadow spread）
- 大圆角堆叠（max 12px）
- 过度渐变
- 信息过满
- Emoji
- 资源站 / 索引站的感觉
