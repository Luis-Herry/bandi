# 追番中心 — 产品需求文档 (PRD)

> 版本：v1.5 | 日期：2026-07-12

## 1. 产品定位

Windows 本地优先追番桌面应用，解决三个核心问题：
1. 今天看什么、看到第几集、下集什么时候更新
2. 每季开头浏览新番、决定追哪些
3. RSS 自动订阅下载，省去手动找资源

视觉定位：个人媒体中心，带有二游官网 / 番剧官网的视觉气质。

## 2. 用户画像

- **主要用户**：项目作者本人（UI/UX 设计师，日常追番）
- **次要用户**：2-3 个朋友（未来开放注册）
- **使用频率**：每天 1-2 次（"今天看什么"），每季集中使用（选番）
- **使用设备**：Windows 桌面应用；核心窗口按桌面宽度和可缩放窗口验收

## 3. 核心场景

### 场景 A：今天看什么（每日高频）
1. 打开首页
2. 3 秒内看到：今天更新了哪些、上次看到哪集、有没有漏看的
3. 对本周新集直接找资源；如果本地已下载，进入内置播放器并恢复播放时间
4. 点击继续观看或标记已看

### 场景 B：新番季初选番（每季集中）
1. 打开番剧库 `/browse`
2. 浏览网格 + 播出日历
3. 半屏预览感兴趣的番
4. 点"追番"设置状态

### 场景 C：偶尔探索
翻角色页 / 声优页，查看出演作品（Phase 2）

## 5. 数据源方案

### Bangumi API（主要）
- 提供中文元数据：番名、封面、集数、播出时间、制作组、声优、评分
- 频率限制：需本地缓存 + 增量同步

### AniList GraphQL API（补充）
- 用于冷门、被禁、国内有争议的番
- 中文化方案：用日文名去 Bangumi 交叉匹配
- 匹配失败时保留日文名，提供手动编辑入口

### RSS 订阅源
- 用户自行配置 RSS 地址
- 匹配规则：番名关键词 + 画质偏好 + 发布组（可选）
- 单集找源需要支持“本地绝对集号”和“发布标题季内集号”的互转，例如本地 EP.74 可匹配字幕组标题里的 `- 08`
- 单集找源过滤合集、前/后半全集、SP/OVA/OAD/NCOP/NCED 包和纯 `Vol.` 卷，避免把合集当成单集候选
- 定时检查 → 提取 magnet/torrent → 以 Safe Mode 推送 qBittorrent

## 6. 下载流程

```
RSS 源（用户配置）
  ↓ node-cron 定时检查
匹配规则筛选
  ↓
提取 magnet / torrent 链接
  ↓
以 Safe Mode 推送 qBittorrent Web API（限制上传 + 分享限制）
  ↓
qBit 自动下载到指定目录
  ↓
应用显示下载状态（等待/下载中/完成/失败）
  ↓
完成下载回写 episodes.isDownloaded，并暂停对应 torrent
  ↓
详情页 / 首页显示内置播放入口
  ↓
内置播放器读取本地文件流并保存 playbackProgress
  ↓
可按需读取外挂字幕或保存截图到 Windows“图片”目录下的 `Bandi` 文件夹
  ↓
达到完成阈值后同步 currentEpisode 和 watchEvents
  ↓
必要时用系统默认外部播放器兜底
```

## 7. 非功能需求

- **性能**：首页加载 < 2s，动效 60fps
- **数据持久化**：SQLite 单文件，无需额外数据库服务
- **安全**：桌面一次性会话令牌只注入当前 Next 端口的 `127.0.0.1` / `localhost` 请求；令牌不落盘、不写日志；内部用户 ID 保持数据关联
- **存储**：数据库、配置、日志、受管 qBit profile 和服务缓存留在 `%APPDATA%\anime-tracker`；截图写入 Windows“图片”目录下的 `Bandi` 文件夹；视频下载目录由用户在首次引导或设置中心选择，默认建议 Windows“视频”目录下的 `Bandi\Downloads`
- **容错**：Bangumi API 不可用时显示本地 fallback 数据 + 提示条；应用启动需通过 `node --use-env-proxy` 继承本机代理；受管 qBit 回环流量强制绕过代理；BT 下载默认限制上传、设置分享限制并在完成后暂停；浏览器无法解码当前视频封装、编码、复杂字幕或多音轨时保留外部播放器入口
- **可用性**：Windows 安装版与 portable 可启动；开发链路通过 `npm run desktop:start` / `npm run desktop:dist` 验证

## 8. 成功标准（Phase 1）

1. 首页加载后 Hero 轮播可用，今日更新和继续观看数据正确
2. 能通过搜索添加番剧（Bangumi API），设置追番状态
3. 番剧详情页完整，可逐集标记"已看"，已下载单集可进入内置播放器并通过外部播放器兜底
4. 我的追番页按状态筛选、排序，网格/列表切换
5. RSS 源可添加/删除，下载队列显示 qBittorrent 状态，单集即时找源不会混入合集或错误季别结果
6. node-cron 定时检查 RSS 并自动推送下载
7. Desktop 静默会话可自动进入本机用户，令牌只在当前回环端口有效，界面不出现账号密码框
8. 封面色提取正常，强调色跟随当前番剧变化
9. 动效平滑，首页灰烬 canvas 与登录页背景保持 60fps 预算
10. Windows 安装版与 portable 可运行，受管 qBit 从 18180 起动态选端口；外部 18080 诊断不会混入受管任务
11. 继续观看能显示内置播放器记录的本集播放时间，并在达到完成阈值后自动同步追番进度
12. 番剧库支持年份筛选和 `YYYY年1月 / 4月 / 7月 / 10月` 季度入口，缩放窗口时季度 tabs 仍能发现完整选项
13. 核心路由在支持的桌面窗口宽度下无页面级横向溢出，标题栏、导航、播放器和下载操作保持可达
14. 动漫本地扫描先预览后写入，跨区路径冲突安全跳过；豆瓣动画优先复用可信 Bangumi 条目，无可信候选时不创建孤儿行
15. 卡片、详情 Hero 和全局搜索三处封面都通过同一载荷校验；缓存与截图只写入明确解析出的 Windows 用户目录
