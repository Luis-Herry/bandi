# 追番中心 — 产品需求文档 (PRD)

> 版本：v1.4 | 日期：2026-06-05

## 1. 产品定位

个人追番管理网站，解决三个核心问题：
1. 今天看什么、看到第几集、下集什么时候更新
2. 每季开头浏览新番、决定追哪些
3. RSS 自动订阅下载，省去手动找资源

视觉定位：个人媒体中心，带有二游官网 / 番剧官网的视觉气质。

## 2. 用户画像

- **主要用户**：项目作者本人（UI/UX 设计师，日常追番）
- **次要用户**：2-3 个朋友（未来开放注册）
- **使用频率**：每天 1-2 次（"今天看什么"），每季集中使用（选番）
- **使用设备**：桌面浏览器为主；当前正在补齐 768px + 375px 响应式适配，核心路由已进入移动端验收，角色/声优资料页仍待适配

## 3. 核心场景

### 场景 A：今天看什么（每日高频）
1. 打开首页
2. 3 秒内看到：今天更新了哪些、上次看到哪集、有没有漏看的
3. 对本周新集直接找资源；如果本地已下载，进入内置播放器并恢复播放时间
4. 点击继续观看或标记已看

### 场景 B：新番季初选番（每季集中）
1. 打开番剧库 `/browse`
2. 浏览网格 + 播出日历
3. 半屏预览感兴趣的番
4. 点"追番"设置状态

### 场景 C：偶尔探索
翻角色页 / 声优页，查看出演作品（Phase 2）

## 5. 数据源方案

### Bangumi API（主要）
- 提供中文元数据：番名、封面、集数、播出时间、制作组、声优、评分
- 频率限制：需本地缓存 + 增量同步

### AniList GraphQL API（补充）
- 用于冷门、被禁、国内有争议的番
- 中文化方案：用日文名去 Bangumi 交叉匹配
- 匹配失败时保留日文名，提供手动编辑入口

### RSS 订阅源
- 用户自行配置 RSS 地址
- 匹配规则：番名关键词 + 画质偏好 + 发布组（可选）
- 单集找源需要支持“本地绝对集号”和“发布标题季内集号”的互转，例如本地 EP.74 可匹配字幕组标题里的 `- 08`
- 单集找源过滤合集、前/后半全集、SP/OVA/OAD/NCOP/NCED 包和纯 `Vol.` 卷，避免把合集当成单集候选
- 定时检查 → 提取 magnet/torrent → 以 Safe Mode 推送 qBittorrent

## 6. 下载流程

```
RSS 源（用户配置）
  ↓ node-cron 定时检查
匹配规则筛选
  ↓
提取 magnet / torrent 链接
  ↓
以 Safe Mode 推送 qBittorrent Web API（限制上传 + 分享限制）
  ↓
qBit 自动下载到指定目录
  ↓
网站显示下载状态（等待/下载中/完成/失败）
  ↓
完成下载回写 episodes.isDownloaded，并暂停对应 torrent
  ↓
详情页 / 首页显示内置播放入口
  ↓
内置播放器读取本地文件流并保存 playbackProgress
  ↓
可按需读取外挂字幕或保存截图到本地 output/player-screenshots
  ↓
达到完成阈值后同步 currentEpisode 和 watchEvents
  ↓
必要时用系统默认外部播放器兜底
```

## 7. 非功能需求

- **性能**：首页加载 < 2s，动效 60fps
- **数据持久化**：SQLite 单文件，无需额外数据库服务
- **安全**：密码 bcrypt 加密，登录 session 管理
- **容错**：Bangumi API 不可用时显示本地 fallback 数据 + 提示条；应用启动需通过 `node --use-env-proxy` 继承本机代理；桌面版 qBittorrent 默认使用 `127.0.0.1:8080`，端口冲突时通过本地配置调整；BT 下载默认限制上传、设置分享限制并在完成后暂停，减少对 v2rayN / TUN 的影响；浏览器无法解码当前视频封装、编码、复杂字幕或多音轨时保留外部播放器入口
- **可用性**：Windows 上 `npm run build && npm start` 直接运行

## 8. 成功标准（Phase 1）

1. 首页加载后 Hero 轮播可用，今日更新和继续观看数据正确
2. 能通过搜索添加番剧（Bangumi API），设置追番状态
3. 番剧详情页完整，可逐集标记"已看"，已下载单集可进入内置播放器并通过外部播放器兜底
4. 我的追番页按状态筛选、排序，网格/列表切换
5. RSS 源可添加/删除，下载队列显示 qBittorrent 状态，单集即时找源不会混入合集或错误季别结果
6. node-cron 定时检查 RSS 并自动推送下载
7. 登录系统可用，未登录重定向到登录页
8. 封面色提取正常，强调色跟随当前番剧变化
9. 动效平滑，首页灰烬 canvas 与登录页背景保持 60fps 预算
10. Windows 上 `npm run build && npm start` 可运行
11. 继续观看能显示内置播放器记录的本集播放时间，并在达到完成阈值后自动同步追番进度
12. 番剧库支持年份筛选和 `YYYY年1月 / 4月 / 7月 / 10月` 季度入口，移动端季度 tabs 不依赖可见滚动条才能发现完整选项
13. 核心路由在 375px / 768px / 1280px 下无页面级横向溢出；角色/声优资料页作为下一轮响应式修复缺口单独跟进
