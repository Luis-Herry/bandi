# 追番中心 — 产品需求文档 (PRD)

> 版本：v1.2 | 日期：2026-06-03

## 1. 产品定位

个人追番管理网站，解决三个核心问题：
1. 今天看什么、看到第几集、下集什么时候更新
2. 每季开头浏览新番、决定追哪些
3. RSS 自动订阅下载，省去手动找资源

视觉定位：个人媒体中心，带有二游官网 / 番剧官网的视觉气质。

## 2. 用户画像

- **主要用户**：项目作者本人（UI/UX 设计师，日常追番）
- **次要用户**：2-3 个朋友（未来开放注册）
- **使用频率**：每天 1-2 次（"今天看什么"），每季集中使用（选番）
- **使用设备**：桌面浏览器（Phase 1），移动端（Phase 2）

## 3. 核心场景

### 场景 A：今天看什么（每日高频）
1. 打开首页
2. 3 秒内看到：今天更新了哪些、上次看到哪集、有没有漏看的
3. 对本周新集直接找资源；如果本地已下载，直接播放
4. 点击继续观看或标记已看

### 场景 B：新番季初选番（每季集中）
1. 打开番剧库 `/browse`
2. 浏览网格 + 播出日历
3. 半屏预览感兴趣的番
4. 点"追番"设置状态

### 场景 C：偶尔探索
翻角色页 / 声优页，查看出演作品（Phase 2）

## 5. 数据源方案

### Bangumi API（主要）
- 提供中文元数据：番名、封面、集数、播出时间、制作组、声优、评分
- 频率限制：需本地缓存 + 增量同步

### AniList GraphQL API（补充）
- 用于冷门、被禁、国内有争议的番
- 中文化方案：用日文名去 Bangumi 交叉匹配
- 匹配失败时保留日文名，提供手动编辑入口

### RSS 订阅源
- 用户自行配置 RSS 地址
- 匹配规则：番名关键词 + 画质偏好 + 发布组（可选）
- 单集找源需要支持“本地绝对集号”和“发布标题季内集号”的互转，例如本地 EP.74 可匹配字幕组标题里的 `- 08`
- 单集找源过滤合集、前/后半全集、SP/OVA/OAD/NCOP/NCED 包和纯 `Vol.` 卷，避免把合集当成单集候选
- 定时检查 → 提取 magnet/torrent → 推送 qBittorrent

## 6. 下载流程

```
RSS 源（用户配置）
  ↓ node-cron 定时检查
匹配规则筛选
  ↓
提取 magnet / torrent 链接
  ↓
带安全下载选项推送 qBittorrent Web API
  ↓
qBit 自动下载到指定目录
  ↓
网站显示下载状态（等待/下载中/完成/失败）
  ↓
完成下载回写 episodes.isDownloaded 并暂停 torrent
  ↓
完成状态的 downloadQueue 记录背书详情页 / 首页本地播放入口
  ↓
用户可只清理追番中心本地列表，qBit 任务和本地文件保留
```

## 7. 非功能需求

- **性能**：首页加载 < 2s，动效 60fps
- **数据持久化**：SQLite 单文件，无需额外数据库服务
- **安全**：密码 bcrypt 加密，登录 session 管理
- **容错**：Bangumi API 不可用时显示本地 fallback 数据 + 提示条；应用启动需通过 `node --use-env-proxy` 继承本机代理；qBittorrent 本机默认端口不可达时可尝试 18080，桌面版引导和默认配置保持 8080
- **可用性**：Windows 上 `npm run build && npm start` 直接运行

## 8. 成功标准（Phase 1）

1. 首页加载后 Hero 轮播可用，今日更新和继续观看数据正确
2. 能通过搜索添加番剧（Bangumi API），设置追番状态
3. 番剧详情页完整，可逐集标记"已看"，已下载单集可直接调起本地播放器
4. 我的追番页按状态筛选、排序，网格/列表切换
5. RSS 源可添加/删除，下载队列显示 qBittorrent 状态，单集即时找源不会混入合集或错误季别结果
6. 下载完成、失败和连接中断会进入顶部通知；下载列表可批量移除本地记录且不删除 qBit 任务或本地文件
7. node-cron 定时检查 RSS 并自动推送下载
8. 登录系统可用，未登录重定向到登录页
9. 封面色提取正常，强调色跟随当前番剧变化
10. 动效平滑，3D 背景 60fps
11. Windows 上 `npm run build && npm start` 可运行；桌面版可通过 `npm run desktop:dist` 打包分发
